## Bảng 8: `dishes`

**Mô tả:** Bảng trung tâm lưu tất cả thông tin cơ bản của món ăn.

```sql
CREATE TABLE dishes (
    id                       VARCHAR(26)  NOT NULL,
    name                     VARCHAR(200) NOT NULL,
    slug                     VARCHAR(250) NOT NULL                   COMMENT 'URL-friendly, auto-gen từ name',
    description              TEXT         NULL,
    main_image_url           VARCHAR(500) NULL,
    cooking_time_minutes     SMALLINT UNSIGNED NOT NULL DEFAULT 0   COMMENT 'Thời gian nấu (phút)',
    prep_time_minutes        SMALLINT UNSIGNED NOT NULL DEFAULT 0   COMMENT 'Thời gian chuẩn bị (phút)',
    servings                 TINYINT UNSIGNED  NOT NULL DEFAULT 1   COMMENT 'Khẩu phần chuẩn',
    difficulty               ENUM('easy','medium','hard') NOT NULL DEFAULT 'medium',
    status                   ENUM('draft','pending_review','published','rejected') NOT NULL DEFAULT 'draft',
    is_nutritionist_verified TINYINT(1)   NOT NULL DEFAULT 0,
    nutritionist_note        TEXT         NULL,
    view_count               INT UNSIGNED NOT NULL DEFAULT 0,
    favorite_count           INT UNSIGNED NOT NULL DEFAULT 0        COMMENT 'Denormalized counter',
    created_by               VARCHAR(26)  NULL                      COMMENT 'admin_users.id',
    reviewed_by              VARCHAR(26)  NULL                      COMMENT 'admin_users.id (nutritionist)',
    reviewed_at              DATETIME     NULL,
    published_at             DATETIME     NULL,
    created_at               DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at               DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at               DATETIME     NULL,
    PRIMARY KEY (id),
    UNIQUE KEY uq_dishes_slug (slug),
    FULLTEXT KEY ft_dishes_name (name),
    CONSTRAINT fk_dishes_created_by FOREIGN KEY (created_by) REFERENCES admin_users(id) ON DELETE SET NULL,
    CONSTRAINT fk_dishes_reviewed_by FOREIGN KEY (reviewed_by) REFERENCES admin_users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**Status Flow:**
```
draft ──► pending_review ──► published
  ▲               │
  └── rejected ◄──┘
```

**Business rules:**
- `status = published` + `deleted_at IS NULL` → hiển thị cho user
- Khi content_editor cập nhật published dish → status → `pending_review`
- `favorite_count` được cập nhật khi INSERT/DELETE trên bảng `favorites`

## Bảng 9: `dish_images`

**Mô tả:** Ảnh bổ sung của món (ngoài `main_image_url`).

CREATE TABLE dish_images (

## Bảng 10: `dish_diet_goals`

**Mô tả:** Bảng junction — mối quan hệ N:N giữa dish và diet_goal enum.

```sql
CREATE TABLE dish_diet_goals (
    dish_id   VARCHAR(26) NOT NULL,
    diet_goal ENUM(
                  'weight_loss','weight_gain','diabetes',
                  'eat_clean','vegetarian','healthy',
                  'low_carb','high_protein'
              ) NOT NULL,
    PRIMARY KEY (dish_id, diet_goal),
    CONSTRAINT fk_ddg_dishes FOREIGN KEY (dish_id) REFERENCES dishes(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**Note:** Mỗi món có thể phù hợp nhiều mục tiêu. Khi generate meal plan, query:
```sql
SELECT d.* FROM dishes d
INNER JOIN dish_diet_goals ddg ON d.id = ddg.dish_id
WHERE ddg.diet_goal = :userDietGoal
  AND d.status = 'published'
  AND d.deleted_at IS NULL
```

## Bảng 12: `dish_allergens`

**Mô tả:** Thành phần gây dị ứng trong món ăn — dùng để lọc khi generate meal plan.

```sql
CREATE TABLE dish_allergens (
    dish_id       VARCHAR(26) NOT NULL,
    allergen_code VARCHAR(50) NOT NULL COMMENT 'Khớp với user_allergies.allergen_code',
    PRIMARY KEY (dish_id, allergen_code),
    CONSTRAINT fk_da_dishes FOREIGN KEY (dish_id) REFERENCES dishes(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**Query lọc allergen khi generate meal plan:**
```sql
-- Loại bỏ món có allergen của user
AND d.id NOT IN (
    SELECT da.dish_id FROM dish_allergens da
    INNER JOIN user_allergies ua ON da.allergen_code = ua.allergen_code
    WHERE ua.user_id = :userId
)
```

## Bảng 13: `dish_nutrition`

**Mô tả:** Thông tin dinh dưỡng chi tiết — tách riêng để dễ cập nhật độc lập.

```sql
CREATE TABLE dish_nutrition (
    id           VARCHAR(26)    NOT NULL,
    dish_id      VARCHAR(26)    NOT NULL,
    calories     DECIMAL(7,2)   NOT NULL DEFAULT 0    COMMENT 'kcal',
    protein_g    DECIMAL(6,2)   NOT NULL DEFAULT 0    COMMENT 'Protein (g)',
    carbs_g      DECIMAL(6,2)   NOT NULL DEFAULT 0    COMMENT 'Carbohydrate (g)',
    fat_g        DECIMAL(6,2)   NOT NULL DEFAULT 0    COMMENT 'Chất béo tổng (g)',
    fiber_g      DECIMAL(6,2)   NOT NULL DEFAULT 0    COMMENT 'Chất xơ (g)',
    sugar_g      DECIMAL(6,2)   NULL                  COMMENT 'Đường (g)',
    sodium_mg    DECIMAL(7,2)   NULL                  COMMENT 'Natri (mg)',
    created_at   DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uq_dn_dish (dish_id),
    CONSTRAINT fk_dn_dishes FOREIGN KEY (dish_id) REFERENCES dishes(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**Tổng dinh dưỡng ngày** được tính khi load meal plan:
```sql
SELECT
    SUM(dn.calories) AS total_calories,
    SUM(dn.protein_g) AS total_protein,
    SUM(dn.carbs_g) AS total_carbs,
    SUM(dn.fat_g) AS total_fat
FROM meal_plan_items mpi
INNER JOIN dish_nutrition dn ON mpi.dish_id = dn.dish_id
WHERE mpi.meal_plan_id = :planId
```
